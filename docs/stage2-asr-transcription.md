# Stage 2 — ASR 转录

> 把音频变成可读文本。

## 目标

把 Stage 1 的音频转成带时间戳的文本(segments),供 Stage 3 清洗、Stage 4 分析使用。最终交付:
- 每个 BV 对应一份 `transcripts/{stem}.json`
- segments 数组,每段含 `start / end / text`

## 双源策略(关键选型)

**两种 ASR 同时用,各有优势**:

| ASR 来源 | 优点 | 缺点 | 场景 |
|---|---|---|---|
| **B 站 AI 字幕**(平台 API) | 中文口语识别**通常优于** whisper-large-v3-turbo;有 "music" 字段可过滤 BGM 段 | 不是所有视频都开了 AI 字幕;混剪视频字幕可能识别错音轨 | **直播回放优先用,效果最好** |
| **whisper.cpp + large-v3-turbo** | 本地跑,稳定;能转录所有音频 | 中文场景下错字率比 B 站 AI 字幕高;速度受硬件限制(Mac M4 上约 5-10x 实时) | **B 站没字幕时退化用** |

### 选型经验

**这是个反直觉发现**:本以为本地的 whisper-large-v3-turbo 会比 B 站 AI 字幕更准——结果实测**在中文直播口语上,B 站 AI 字幕反而更准**。可能原因:
- B 站 AI 字幕模型针对**中文 + 短句 + 口语化**做过优化
- whisper-large-v3-turbo 是多语言通用模型,中文不是它最强的语种
- 直播口语含大量语气词 / 拖音 / 口癖,通用 ASR 容易识别成噪音

**实际策略**:
1. 先尝试 B 站 player API 拿 AI 字幕(JSON 格式,可直接转 segments)
2. 没字幕的视频再 fallback whisper.cpp 本地转录
3. **跑完之后必须抽样人工验证**(见下方"切片字幕的污染发现")

## B 站 AI 字幕的获取

- 通过 `https://api.bilibili.com/x/player/v2?bvid={bvid}&cid={cid}` 拿到字幕 URL
- 找 `lan` 字段以 `ai-` 开头的(如 `ai-zh`)
- 下载字幕 JSON,过滤 `music > 0.5` 的段(BGM / 唱歌段,不是说话)
- 转成项目通用的 segments 格式

需要 B 站 cookie(确保有访问权限)。本项目用 `browser_cookie3` 从 Chrome 读 cookie。

## 切片字幕的污染发现(关键教训)

项目早期为了扩样本量,**额外下载了 944 个粉丝切片**的字幕。这造成了一个**结构性数据问题**:

### 现象

跑完 Stage 4 LLM 风格分析后,跨场聚合统计发现一些**怪异的高频词**——比如"分体水冷""LOL 比赛术语""怀石料理"等。一开始以为是 LLM 归纳出问题,后来抽查发现:**这些词来自切片字幕本身**,根本不是奶绿说的。

### 根因

**粉丝切片的视频形态导致**:

```
00:00 - 02:00  播放被看的源视频(分体水冷教程 / LOL 比赛 / 旅游 vlog 等)
              主声道:源视频音频(大且清晰)
              背景声:主播小声评论(信号弱)

02:00 - 03:00  主播单独发言(主声道终于是主播)
```

B 站 AI 字幕系统**只识别主声道**,主声道大部分是源视频音轨——所以字幕主要转录的是**被看视频的内容**,不是主播的反应。

### 影响范围

抽查发现 944 切片里有 **113 个(12%)** 字幕**根本不是主播**,占总段数的 **27.5%**(脏切片平均 431 段/个,是干净切片的 3 倍)。

举例:
- `BV1ye411r7xn`("【明前奶绿】大凶坏女人立体声训猫")字幕实际是分体水冷教程
- `BV1ENXXBLE8T`("【明前奶绿/自用】我是你的赛博妈妈")字幕实际是 LOL 比赛解说
- `BV1Jq9HYiEnE`("店员犯错店长肉偿")字幕实际是改革开放讲解

### 修复

**944 切片全部归档**(包括 12% 明显脏的 + 88% 难以验证质量的)。**Stage 4 只用 14 场直播回放**——主播全程单一发言,字幕基本干净。

### 教训

1. ASR 阶段**不能只看字幕能不能拿到**,要验证**字幕内容是不是想要的人在说**
2. 粉丝切片对"风格归纳"价值低——粉丝剪辑通常保留源材料(她在看什么),不是她在说什么
3. 单一发言型(纯独白 / 念稿)的切片字幕才干净,但这类切片少
4. 验证方法:抽样读字幕开头几段,看内容是否匹配标题/主题。**不匹配 = 字幕被劫持**

## 不公开的内容

- 944 切片字幕(脏 + 干净都不发布,因为单条字幕也属于直播内容)

- B 站 API 调用脚本(避免被滥用扫主播)

## 给后来者的话

- 中文直播 ASR 不要默认 whisper 比平台 AI 字幕好——实测对比
- 字幕拿到后**第一件事是抽样人工读**——不要急着喂下游
- 切片字幕慎用,**反应型切片字幕几乎不可用**(她不是声音主体)
- 模型选型上,SenseVoice 在中文场景理论上比 whisper-large 强,但仍可能受"识别最大声音轨"影响——同样要人工抽检
