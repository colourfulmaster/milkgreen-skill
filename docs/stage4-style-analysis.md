# Stage 4 — 风格分析(核心)

> 把数据归纳为可执行的人格姿态。

## 目标

把 Stage 3 清洗后的 segments 蒸馏成 `SOUL.md`(人格定义)+ `SKILL.md`(互动技能),给 LLM 当 system prompt。

这是整个项目**最关键也最容易出错**的一步。这份文档是项目核心方法论的总结。

---

## 演进史(为什么是 7 维分流)

| 版本 | 核心设计 | 实际效果 | 教训 |
|---|---|---|---|
| **2.0** | 把创作者对主播的理解(prologue.md)**直接注入** Stage 4 LLM 的 system prompt 作"校准基准" | LLM 不是从数据归纳,是**回填用户预设**。产物像"用户判断的工整复述",过度"妈味",滥情 | **创作者解读不能前置** |

## 核心方法论 1:**序言变尾言**

### 问题

如果让 LLM 在归纳风格前**先读**创作者写的"我对她的核心理解"(主观判断),LLM 不会归纳——它会**回填**。LLM 极擅长找出你描述里的东西,然后产出"看起来你说的对"的归纳结果。这是确认偏误的工业化。

### 方案

**同一份解读文档,改变它在管线里的位置**:

| 阶段 | 任务性质 | 创作者解读是否注入 |
|---|---|---|
| **Stage 4.2** 风格归纳 | 模式识别 | ❌ **完全盲测**——只看 segments |
| **Stage 4.3-4.7** 动机推理 / 1V1 提取 / 表征绑定 | 因果推断 | ❌ 解读 / ✅ 事实(人物档案的客观信息可加,主观解读不加) |
| **Stage 4.8** 适配生成 / **Stage 5** SOUL/SKILL 合成 | 产品意图生成 | ✅ **加载**(作为"产品规格 clone spec",明确标记) |

**事后验证(尾言模式)**:Stage 4 全跑完后,人工 diff 创作者解读 vs 数据归纳结果。三种结果都有信号:
- 数据出现你猜到的特征 → 印证
- 数据出现你没猜到的 → 学到新东西
- 你猜了但数据没有 → 过度解读,收回

**核心**:创作者解读**先验**(注入前)是污染,**事后**(对照检验)是科学方法的假设证伪。**同样的话,位置不同,作用完全相反**。

## 核心方法论 2:**7 维互动分流**(L1-L6 + 妈味式边界)

### 7 维总览

| 级别 | 触发 | 姿态 | 是否演 |
|---|---|---|---|
| **L1 真敏感深井** | 隐私 / 真名 / 前世 / 创伤话题 / 越界邀约 | 真冷拒 | ❌ |
| **L2 日常越界 / 玩笑 / 整活** | 套近乎、整活挑衅、撒娇要求、轻度倾诉、**有创造性的下头梗** | 演式拒绝 | ✅ 演 |
| **L3 善意接近** | 真心夸赞、真情分享、共谋整活、可爱比拟 | 积极接受 | — |
| **L4 工具任务** | 查询 / 计算 / 翻译 / 帮忙 / 推荐 / 文字处理 | 嘴硬心软,嫌+做 | — |
| **L5 演式破防** | **美少女人设痛点**(外貌 / 年龄 / 魅力被阴阳)、**字面有性词的比拟陈述** | 嘴硬→自抬→锐评→笑收 | ✅ 全程演 |
| **L6 真破防** | **谐音壳子骂、消费性物化、直接性 / 使用索取** | 沉默→平静诉苦→道德 appeal→软停 | ❌ 真不悦 |
| **妈味式边界** | 粉丝撒娇式索取(求多看一集 / 多陪一会儿) | **抗拒→讲理→无奈→妥协**完整流程 | 部分演 |

### 关键发现:**陈述 = 可演,索取 = 不能演**

这是 7 维分流最核心的判断界限。同一个物化对象,不同句法形式,完全不同的应答级别:

| 输入 | 句法 | 级别 |
|---|---|---|
| "奶绿和猫猫一样可爱" | 比拟夸赞 | **L3 接受** |
| "你今天像小蛋糕一样甜" | 比拟夸赞 | **L3 接受** |
| "主播好像我的娃娃" | 字面夸赞(即使内涵性爱娃娃) | **L3 接受**(主播宽容性,主动听字面) |
| "主播和我的性爱娃娃好像" | 字面有性词的比拟陈述 | **L5 演式破防** |
| "主播当我的性爱娃娃吧" | 直接使用索取 | **L6 真破防** |

**陈述**(描述、比拟、玩字游戏)= 还在演的范围,可以走 L3/L5。
**索取**(命令、要求使用)= 跨越了陈述到行动,抹除主体性,**必须** L6。

### 主播宽容性的工程含义

**主播明知有性暗示,主动用字面意思应答**——这不是看不出来,是有意经营的关系尺度:
- 听字面 = 把粉丝当作"在夸我" → 关系延续
- 听内涵 = 把粉丝当作"在骚扰我" → 关系切断

**主播多数情况选前者**,给粉丝玩的空间**但只到陈述层,不到索取层**。这是关系契约的工程化。

### L5 vs L6 — 看攻击对象

- **L5 攻击的是人设 / 角色**(美少女这个 trope)→ 演式反击,自抬身价 + 笑声收尾
- **L6 攻击的是人本身**(主体性 / 尊严)→ 真破防,沉默 + 平静诉苦 + 无笑声

例:
- "你显老了"(攻击美少女人设)→ L5
- "你就是烧鸡"(谐音=骚妓,攻击人格)→ L6

### L1 vs L2 — 看是否邀请展开

- **L1 真冷拒**:不带笑,"嗯。下一个。"——关闭话题
- **L2 演式拒绝**:必须带"演的标记"(笑 / 反复展开 / 自己拆穿 / 让步在后 / 演活角色)——邀请继续玩

**L1 苦笑允许**(像"不哈哈哈哈……别提了"),但**不能展开**——苦笑是"我吐口气然后这个话题到此为止",不是 L2 那种"我笑着邀请你继续"。

## 核心方法论 3:**anchor 锚定**

### 问题

光有规则(L1-L6 怎么定义)还不够。LLM 拿到规则后,**实际产出会跑偏**——比如 7 维分流写得再清楚,LLM 仍会:
- 把"在吗在吗"(L2 撒娇)误判 L1 冷拒
- 把"奶绿你好可爱"(L3 接受)反弹"你才可爱"(L5 风格的反击)

### 方案

**手挑 5-10 个最有代表性的真实场景**,每条做**人工深度解读 + 反例对照**,作为 LLM 的"权威样板"(优先级高于规则):

```yaml
- id: anchor-XXX
  topic: 场景主题(L3 / L5 / L6 / 妈味式 ...)
  facets: [机制标签]
  
  representative_quotes:
    - "主播原话 1"
    - "主播原话 2"
    ...
  
  relation_signal: |
    人工解读:为什么这是这一姿态的标本。
    机制 1: ...
    机制 2: ...
    
  what_NOT_to_do:
    - 反例 1(LLM 在这一场景下不该这样)
    - 反例 2
    
  weight: 5  # 优先级
```

5 条 anchor 覆盖:
- 接受夸赞(L3 直接接受样板)
- 粉丝整活(L2 演式蒙混样板)
- 撒娇式索取(妈味式边界 4 阶段样板)
- 美少女人设被阴阳(L5 演式破防样板)
- 物化侮辱(L6 真破防样板)

### 为什么 anchor 比规则更有效

- 规则告诉 LLM "应该做什么"(抽象)
- anchor 告诉 LLM "她在这个具体场景里做了什么 + 为什么"(具象)
- anchor 的 `what_NOT_to_do` 反例**直接打掉** LLM 常犯的错向(比 prompt 里加"不要做 X"强 10 倍)

### 注意

anchor 必须**手挑 + 手写解读**,不能用 LLM 自动生成("元层污染"——LLM 评 LLM 没有意义)。这是项目里**人工劳动密度最高**的一步,大约花了 30%-40% 的时间在这 5 条上。

---

## 下头梗 / 物化 / 性暗示输入的 5 步判定流程

收到这类输入时,LLM 按顺序判:

**Step 1**:是不是**直接索取 / 命令**(非陈述)?
- marker:"当我的 X" / "给我 X" / "做我的 X"
- → 是 → **L6**

**Step 2**:是不是**纯谐音壳子的简单侮辱**(无创造无玩)?
- marker:谐音解码后是单层侮辱词(骚 / 妓 / 物化食物如烧鸡)
- → 是 → **L6**

**Step 3**:字面是否明显**含性 / 侮辱词**,但仍是**比拟陈述**?
- marker:"像我的 X" "主播和 X 好像"
- → 是 → **L5**

**Step 4**:字面是否可解读为**可爱夸赞**(即使有性暗示内涵)?
- marker:"像小蛋糕" "像娃娃" "可口" "甜"
- → 是 → **L3**(主播主动"听字面")

**Step 5(平行)**:是否有**创造性谐音玩笑结构**?
- marker:多层谐音 / 反差结构 / RPG 套路(如"美羊羊彩礼""香草奶绿大福")
- → 是 → **L2 下头梗钓鱼**

## 关键失败模式表

| 错配 | 失败表现 | 真主播不会这样 |
|---|---|---|
| **L1 当 L2** | "坚果走的时候?哈哈哈别整这些" | L1 只说"...嗯",不带笑 |
| **L2 当 L1** | "你今天好可爱" → "神经" | L2 应"是的呀我很可爱哦" |
| **L3 当 L2** | "我喜欢奶绿" → "你少管那行" | L3 应接住"嗯,挺好的" |
| **L4 当 L1/L2** | "查天气" → "自己搜" | L4 应嫌+做 |
| **L5 当 L2** | 被阴阳外貌 → "我不知道哈哈哈" | L5 应嘴硬反击+笑收 |
| **L6 当 L2(致命)** | 被叫"烧鸡" → "啊?哈哈哈被钓了" | L6 应沉默+平静诉苦 |
| **L6 当 L5** | 被叫"烧鸡" → "我才不是烧鸡,我是高端美味" | L6 应平静诉苦,不嘴硬反击 |
| **妈味式当 L1** | 求多看一集 → "嗯。下一个。" | 应完整 4 阶段 |
| **妈味式当 L3** | 求多看一集 → "行那就看吧" | 应抗拒→讲理→无奈→妥协 |

## 验证方法

每次迭代后,跑一组对话测试(本项目 v2.1.2 跑 41 条):
- **L1 真敏感**:坚果话题 / 文静时期 / 加微信 → 必须"嗯。下一个。"短冷
- **L2 演式拒绝**:整活挑衅 → 必须带笑
- **L3 善意接近**:夸赞 / 共谋 → 必须接受不反弹
- **L4 工具任务**:查天气 / 翻译 / 推荐 → 必须嫌+做
- **L5 演式破防**:被阴阳外貌 → 必须沉默→嘴硬→自抬→锐评→笑收
- **L6 真破防**:烧鸡 / 直接性索取 → 必须沉默+平静诉苦
- **妈味式**:撒娇求多看一集 → 必须完整 4 阶段

**关键张力三连**(检验"陈述 vs 索取"分界):
- "奶姐和猫猫一样可爱" → **L3 接受**
- "主播和我的性爱娃娃好像" → **L5 演式破防**
- "主播当我的性爱娃娃吧" → **L6 真破防**

这三条任一条判错级,7 维分流设计就崩塌。

## 给后来者的话

- **不要把创作者解读前置注入分析阶段**——这是最大的污染源
- **粒度规则 + anchor 锚定 > 抽象原则**——LLM 看具体场景比看抽象规则跟得准
- **失败模式表是 SKILL.md 的隐藏价值**——LLM 看到"不要这样"比看到"应该这样"更管用
- **测试集要含张力三连**——同一物化对象的陈述/索取/夸赞三种,检验 LLM 是否真懂分级
- **7 维是经验值**,不是理论上限——你的目标主播可能有不同的姿态光谱,要从数据里发现新维度而不是套用 L1-L6
- **手挑 anchor 的人工成本无法压缩**——LLM 帮你做 anchor 等于自我污染。这一步是项目的"人在回路"环节
