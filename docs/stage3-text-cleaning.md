# Stage 3 — 文本清洗

> 把噪音清掉,把风格留下。

## 目标

对 Stage 2 的 ASR 转录做清洗,产出**信号密度高、风格语气词保留**的 segments,供 Stage 4 LLM 分析。

输入:`transcripts/{stem}.json`(每段 `start / end / text`)
输出:`cleaned/{stem}.json`(同结构 + 情绪标注 + 元信息)

## 清洗步骤

### 1. 过滤纯噪音(但保留风格语气词)

这是 Stage 3 **最容易做错的一步**。

**纯噪音示例**(可删):
- 连续 4+ 个叹词:"啊啊啊啊啊" / "嗯嗯嗯嗯嗯嗯"
- 纯标点 / 纯空白
- 纯数字段(通常是 ASR 把背景音识别成数字)

**风格语气词**(必保留,**这是奶绿声音的核心**):
- 单字:嗯 / 啊 / 哦 / 呃 / 嘛 / 呀 / 吧 / 哈
- 短语:哎呀 / 哎哟 / 我去 / 绝了 / 牛逼 / 卧槽 / 我靠 / 妈呀 / 天哪
- 笑声:哈哈哈 / 嘿嘿 / 嘻嘻
- 损友梗:好家伙 / 真行 / 牛的 / 难绷 / 绷不住 / 没绷住 / 下头 / 逆天 / 抽象
- 短肯定:对对对 / 好好好 / 行吧 / 是的 / 对的 / 没错

**错误教训**:早期版本把"嗯""啊"等单字直接当噪音过滤——结果 Stage 4 LLM 完全归纳不出"她说话会带嗯"这个核心口癖。**风格语气词必须有白名单豁免**。

```python
STYLE_INTERJECTIONS = {
    "嗯", "啊", "哦", "呃", "嘛", "呀", "吧", "哈",
    "哎呀", "哎哟", "我去", "绝了", "牛逼",
    "对对对", "好好好", "行吧", "是的", "对的",
    "哈哈哈", "嘿嘿",
    "卧槽", "我靠", "妈呀", "天哪",
    "好家伙", "真行", "牛的",
    "难绷", "绷不住", "下头", "逆天", "抽象",
    # ...
}
```

### 2. 合并碎片

低于 2 秒**且**少于 5 字的段视为碎片,合并到相邻句(优先前句)。

例外:风格语气词单独成段时**不合并**(它们短但有意义)。

### 3. 连续重复去重

相邻两段相似度 > 0.85(SequenceMatcher)视为重复——保留更长版本。

ASR 在主播停顿时会重复识别("不看不看" → "不看不看,不看,不看" 之类)——去重保持句子干净。

### 4. 基础规范化

- 多余空格合并
- 首尾空白去除
- 保持中文标点 / 英文标点原样(不强行转换)

**不做**:标点恢复。Whisper 中文输出基本没标点,Stage 3 不补——让 Stage 4 LLM 处理标点的事(因为 LLM 在归纳时本来就要看上下文,标点对它不是大问题)。

### 5. 情绪标注(SenseVoice 集成)

用 SenseVoice 对每段标注情绪 / 声音事件。开放标签集合,常见值:
- 中性/平静、开心/兴奋、悲伤/低落、愤怒/激动
- Speech、Laughter、SURPRISED、Cough、DISGUSTED
- EMO_UNKNOWN(识别不出来时)

存到 `segments[].emotion` 字段,Stage 4 可用。

## 输入白名单(数据卫生)

经历了 Stage 1 的 `bili_ai_*` 测试残留污染教训后,在 Stage 3 加了输入白名单:

```python
def is_target_stem(stem: str) -> bool:
    """只接受奶绿真实数据命名。"""
    if stem.startswith("BV") or stem.startswith("clip_BV"):
        return True
    if "_BiliBili_" in stem:
        return True
    return False
```

非命名规范的文件(如 `bili_ai_P1.json` / `capture_xxx.json` 等开发测试残留)被静默 SKIP,日志告警,不进入正式数据流。

**这个白名单后来挡了 1 个之前漏掉的 `capture_*` 测试文件**——说明日常开发里测试产物的污染**是真实风险**,不是想象的。

## 没做的(2.2 候选)

- **PII 脱敏** — 主播念 SC 时会读出粉丝用户名 / ID,这些是 PII。**任何公开发布前必须先脱敏**(本仓库不发字幕所以暂时没做)。如果将来公开数据样本,必做:用 hash 替换用户名,正则识别 `@xxx` 模式等
- **标点恢复** — Whisper 中文输出无标点,Stage 4 LLM 看的是连续字符串,影响句式归纳精度。可加 `ct-punc` 或类似轻量模型
- **场景分类**(开场/收尾/唱歌/读SC/锐评/游戏)— 当前在 Stage 4.1 用正则打,长远应该下沉到 Stage 3 作为 cleaned 文件的字段
- **弹幕-语音配对** — 项目原计划做但没启动(因为没采集弹幕,见 Stage 1 弹幕策略)

## 不公开的内容

- `cleaned/*.json` 全部(含主播原话,版权)
- 清洗脚本代码(本仓库不发代码)

## 给后来者的话

- **保风格语气词不是可选项,是必须**——它们是 LLM 学习口癖的唯一证据
- 测试产物**单独目录**,不要让它跟正式数据混
- 加白名单是廉价的防护——10 行代码,挡掉一整类问题
- 情绪标注不用全做,Stage 4 跨场聚合时主要看高强度情绪(开心/愤怒/悲伤),低强度(中性/平静)价值有限
